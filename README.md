# uds-tender-crm

Внутренняя CRM / система контроля входящих заявок и тендеров для компании УДС-Проект.

Система не должна быть «ещё одной таблицей». Её задача — управлять процессом обработки входящей возможности: создать заявку, согласовать участие, завести обращение и рабочую папку, запустить параллельные задачи, собрать затраты и документы, согласовать КП, подать предложение и зафиксировать результат.

## Главная цель

Понять по каждой заявке:

- где она находится сейчас;
- кто должен выполнить следующее действие;
- какой срок установлен;
- есть ли просрочка;
- сколько времени занял каждый этап;
- почему по заявке участвуем, не участвуем, проиграли или победили.

## Пользователи MVP

На старте активных пользователя два:

- Денис — руководитель процесса обработки заявки;
- Катя — коммерческий директор / ответственная за коммерческую часть, КП и подачу.

ГД, МЛ, юристы, РП/ГИП и заказчики на первой версии учитываются как внешние участники без входа в систему.

## Предлагаемый стек

Для первой версии рекомендуется стек:

- Next.js;
- TypeScript;
- Supabase для базы данных и авторизации;
- простая ролевая модель: `admin`, `user`;
- хранение ссылок на рабочие файлы без загрузки самих документов в систему.

## Что не входит в MVP

В первую версию не входят: интеграция с почтой, интеграция с 1С, автоматический парсинг тендерных площадок, автоматическое создание папок на сервере, автогенерация КП и сложная система прав доступа.
