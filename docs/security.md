# Правила безопасности и работы с данными

## 1. Репозиторий

Репозиторий должен оставаться приватным.

В репозиторий нельзя добавлять:

- реальные коммерческие предложения;
- реальные расчёты затрат;
- проекты договоров;
- протоколы разногласий;
- выгрузки с внутренними данными;
- пароли;
- ключи доступа;
- файлы `.env` с реальными значениями.

## 2. Тестовые данные

Для разработки использовать обезличенные тестовые заявки.

Можно использовать реалистичную структуру, но без реальных:

- заказчиков;
- сумм;
- ссылок;
- договорных условий;
- персональных данных;
- внутренних комментариев.

## 3. Рабочие файлы

В MVP система хранит только ссылки на рабочие материалы.

Файлы остаются в рабочей файловой структуре компании. В систему не загружаются:

- расчёты;
- КП;
- договоры;
- протоколы;
- подтверждения подачи.

## 4. Переменные окружения

Файл `.env.local` не должен попадать в репозиторий.

В репозиторий можно добавить только `.env.example` с пустыми или демонстрационными значениями.

## 5. Права доступа MVP

Минимальные роли:

- admin — полный доступ;
- user — рабочий доступ без изменения системных настроек.

В первой версии не нужно делать сложную матрицу прав. Важно не усложнить внедрение.

## 6. Аудит

Все ключевые действия должны фиксироваться:

- создание заявки;
- смена статуса;
- создание задачи;
- закрытие задачи;
- возврат на доработку;
- изменение срока;
- изменение суммы КП;
- фиксация результата.

Аудит нужен не для контроля ради контроля, а чтобы понимать, где теряется время и почему процесс застревает.

## 7. Демо-хранилище GitHub Pages

Демо-версия на GitHub Pages работает без Supabase, API routes и серверных действий. Все изменения пользователя сохраняются в браузере в `localStorage`.

Заявки, созданные во время выполнения демо, нельзя открывать через статические URL вида `/requests/[id]`: GitHub Pages содержит только HTML-файлы, созданные на этапе static export для моковых заявок. До подключения базы данных карточки runtime-created заявок открываются клиентски на странице `/requests` через параметр выбранной заявки, например `/requests?selectedRequestId=<id>`.

Кнопка «Сбросить демо-данные» очищает известные ключи демо-хранилища в `localStorage` и возвращает интерфейс к начальному набору обезличенных данных.
