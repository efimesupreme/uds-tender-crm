# Экраны MVP

## 1. Дашборд

Назначение: утром быстро понять, где горит процесс.

Блоки:

- задачи на сегодня;
- просроченные задачи;
- заявки без следующего действия;
- ближайшие сроки подачи;
- активные заявки по статусам;
- КП на согласовании у МЛ;
- заявки в ожидании обратной связи;
- причины отказов;
- средние сроки ключевых этапов.

## 2. Реестр заявок

Колонки первой версии:

- ID;
- наименование;
- заказчик;
- регион;
- статус;
- решение об участии;
- срок подачи;
- следующее действие;
- ответственный;
- просрочка;
- сумма КП;
- результат.

Фильтры:

- статус;
- решение об участии;
- ответственный;
- заказчик;
- регион;
- вид работ;
- срок подачи;
- просрочка;
- причина неучастия;
- результат.

## 3. Карточка заявки

Карточка должна быть главным рабочим экраном.

Блоки:

### Шапка

- название;
- заказчик;
- текущий статус;
- срок подачи;
- следующее действие;
- ответственный;
- признак просрочки.

### Базовые данные

- источник;
- ссылка;
- регион;
- тип обращения;
- вид работ;
- краткое описание;
- комментарий.

### Решение об участии

- дата запуска согласования;
- дата решения;
- итог;
- причина неучастия;
- комментарий.

### Обращение и папка

- номер обращения;
- ссылка на рабочую папку;
- дата создания папки;
- статус задачи.

### Затраты

- ответственный;
- срок;
- дата получения;
- статус проверки;
- дата утверждения;
- сумма затрат;
- сумма КП;
- плановая маржа;
- ссылка на расчёт;
- возвраты.

### Договор и протокол

- наличие проекта договора;
- статус анализа юристами;
- дата получения анализа;
- ключевые риски;
- нужен ли протокол;
- статус протокола;
- ссылка на файл.

### Документы

- статус комплекта;
- что не хватает;
- дата готовности;
- комментарий.

### КП и подача

- дата передачи затрат Кате;
- дата подготовки КП;
- сумма КП;
- дата направления на согласование МЛ;
- дата согласования МЛ;
- дата подачи;
- способ подачи;
- ссылка на подтверждение.

### Результат

- результат;
- дата результата;
- причина проигрыша;
- цена победителя;
- наша цена;
- вывод на будущее.

### Задачи

Таблица задач по заявке: тип, исполнитель, срок, статус, факт, просрочка.

### Ссылки

Список рабочих ссылок: папка, затраты, договор, протокол, КП, подтверждение подачи, обратная связь.

### Журнал событий

Хронологическая лента действий.

## 4. Мои задачи

Блоки:

- новые задачи;
- задачи в работе;
- просроченные задачи;
- задачи на сегодня;
- завершённые задачи.

Фильтры:

- заявка;
- тип задачи;
- статус;
- срок.

## 5. Справочники

Редактируемые справочники:

- статусы заявки;
- типы задач;
- причины неучастия;
- причины проигрыша;
- виды работ;
- источники обращений;
- внешние исполнители;
- способы подачи.

## 6. Настройки

Минимально:

- пользователи;
- роли;
- базовые параметры системы;
- правила отображения сроков.
