## 1. Дашборд

Назначение: утром быстро понять, где горит процесс.

Блоки:

- задачи на сегодня;
- просроченные задачи;
- заявки без следующего действия;

- КП на согласовании у МЛ;
- заявки в ожидании обратной связи;
- причины отказов;
- средние сроки ключевых этапов.

### Канбан-представление

Канбан на странице «Заявки» является альтернативой табличному реестру и показывает процесс обработки по укрупнённым стадиям:

- «Входящие» — новые заявки;
- «На решении ГД» — заявки на решении об участии;
- «Заведение обращения» — согласованное участие и заведение обращения/папки;
- «Подготовка материалов» — подготовка, получение и внутреннее согласование материалов и затрат;
- «КП и согласование» — подготовка КП, согласование у МЛ и готовность к подаче;
- «Подано» — отправленные КП;
- «Обратная связь» — ожидание ответа заказчика;
- «Закрыто» — финальные статусы, только для просмотра.

В заголовке каждой колонки отображаются название стадии, количество заявок и сумма КП по заявкам этой колонки. Если сумма КП у заявки не заполнена, она считается как 0.

Карточка заявки содержит ID/внутренний номер, название, заказчика, текущий статус, срок подачи, сумму КП, следующее действие, ответственного и мягкую подсветку проблемных/просроченных заявок.

Перетаскивание карточек между колонками выполняет смену статуса через существующие правила процесса. Для укрупнённых стадий используется основной целевой статус колонки. Колонка «Закрыто» не принимает перетаскивание, потому что закрытие требует отдельной причины или результата. Если переход запрещён процессом, статус не меняется и пользователь видит сообщение: «Переход из текущего статуса в выбранную колонку не разрешён процессом».

### Результат и закрытие

В карточке заявки есть блок **«Результат и закрытие»**. Он показывает текущий результат, дату закрытия, причину, комментарий, нашу цену и цену победителя.

Для активной заявки в этом же блоке доступна простая форма без модальных окон:

- выбор финального статуса;
- выбор причины из справочника выбранного статуса;
- комментарий;
- наша цена;
- цена победителя;
- дата результата;
- кнопка «Закрыть заявку».

Если обязательные поля не заполнены, заявка не закрывается, а пользователь видит понятную ошибку. Колонка канбана «Закрыто» остаётся обзорной: перетаскивание в неё запрещено, закрытие выполняется только через форму результата в карточке заявки.

## Расширенная карточка заявки

Карточка заявки стала рабочим центром обработки заявки. Помимо сводки, статусов, задач, событий и рабочих ссылок в ней ведутся отдельные блоки процесса: решение об участии, затраты, договор и протокол, документы для подачи, КП и подача, обратная связь. Каждый блок показывает текущие значения, редактируется простой встроенной формой без модальных окон и при сохранении пишет отдельное событие в журнал.

### Создание и открытие заявки в демо на GitHub Pages

На странице «Заявки» встроенная форма заменена на кнопку **«Новая заявка»**. По нажатию открывается простая панель создания. Обязательные поля отмечены звёздочкой:

- наименование;
- заказчик;
- регион;
- вид работ;
- ответственный;
- источник.

Если пользователь пытается создать пустую или неполную заявку, интерфейс показывает сообщение «Заполните обязательные поля», перечисляет незаполненные поля и подсвечивает их. После успешного создания панель закрывается, форма очищается, заявка сохраняется в `localStorage` и сразу открывается в карточке.

Карточка заявки открывается из таблицы и канбана внутри страницы `/requests` через клиентский параметр `selectedRequestId`. Это позволяет открывать не только заранее собранные моковые заявки, но и заявки, созданные в браузере во время демо. Если выбранный ID не найден в текущем демо-хранилище, пользователь видит сообщение «Заявка не найдена», а не 404.

## Переключатель пользователя

В правом верхнем углу основного интерфейса расположен простой demo-переключатель «Текущий пользователь». В нём доступны Денис и Катя без паролей, регистрации и настоящей авторизации. Выбор сохраняется в `localStorage` и определяет автора действий в журнале событий.

## Мои задачи по текущему пользователю

Страница «Мои задачи» фильтрует рабочий список по выбранному пользователю. Для Дениса показываются назначенные ему задачи, задачи без системного ответственного и управленческие проблемные заявки. Для Кати показываются назначенные ей задачи, а также задачи по КП, документам, подаче и согласованию МЛ.

## Рабочий фокус на дашборде

На дашборде есть блок «Рабочий фокус». Для Дениса он показывает управленческие риски: заявки без решения об участии, без обращения или рабочей папки после согласования, просроченные задачи, узкие места, близкие сроки подачи и отсутствие следующего действия. Для Кати блок показывает её коммерческую зону: КП в работе, КП у МЛ, документы для подачи, готовность к подаче и поданные заявки, где нужна обратная связь.

## Фильтр «Моя зона» в заявках

На странице «Заявки» рядом с поиском и статусом есть фильтр «Все заявки / Моя зона». Для Дениса «Моя зона» включает его заявки и заявки с управленческими проблемами. Для Кати — заявки, где она ответственная или есть работа по КП, документам, согласованию МЛ, подаче либо обратной связи после подачи.

## Предлагаемые действия на дашборде

На дашборде добавлен блок «Предлагаемые действия». Он показывает рассчитанные автоматические предложения по всем активным заявкам с учётом выбранного demo-пользователя в правом верхнем углу.

Для Дениса выводятся управленческие и процессные подсказки: обращения и рабочие папки, просрочки, запуск следующего этапа. Для Кати выводятся подсказки по КП, согласованию МЛ, подаче, обратной связи и документам. Если у предложения есть автоматическое действие, в строке показана кнопка создания задачи.

## Предлагаемые действия по заявке

В карточке заявки добавлен блок «Предлагаемые действия по заявке». Он показывает только предложения по текущей заявке: текст, причину, приоритет и кнопку действия, если система может создать задачу автоматически.
