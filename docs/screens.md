# Экраны MVP

## 1. Дашборд

Назначение: утром быстро понять, где горит процесс.

Блоки:

- задачи на сегодня;
- просроченные задачи;
- заявки без следующего действия;
- ближайшие сроки подачи;
- активные заявки по статусам;
- КП на согласовании у МЛ;
- заявки в ожидании обратной связи;
- причины отказов;
- средние сроки ключевых этапов.

### Канбан-представление

Канбан на странице «Заявки» является альтернативой табличному реестру и показывает процесс обработки по укрупнённым стадиям:

- «Входящие» — новые заявки;
- «На решении ГД» — заявки на решении об участии;
- «Заведение обращения» — согласованное участие и заведение обращения/папки;
- «Подготовка материалов» — подготовка, получение и внутреннее согласование материалов и затрат;
- «КП и согласование» — подготовка КП, согласование у МЛ и готовность к подаче;
- «Подано» — отправленные КП;
- «Обратная связь» — ожидание ответа заказчика;
- «Закрыто» — финальные статусы, только для просмотра.

В заголовке каждой колонки отображаются название стадии, количество заявок и сумма КП по заявкам этой колонки. Если сумма КП у заявки не заполнена, она считается как 0.

Карточка заявки содержит ID/внутренний номер, название, заказчика, текущий статус, срок подачи, сумму КП, следующее действие, ответственного и мягкую подсветку проблемных/просроченных заявок.

Перетаскивание карточек между колонками выполняет смену статуса через существующие правила процесса. Для укрупнённых стадий используется основной целевой статус колонки. Колонка «Закрыто» не принимает перетаскивание, потому что закрытие требует отдельной причины или результата. Если переход запрещён процессом, статус не меняется и пользователь видит сообщение: «Переход из текущего статуса в выбранную колонку не разрешён процессом».

## 3. Карточка заявки

Карточка должна быть главным рабочим экраном.

Блоки:

### Шапка

- название;
- заказчик;
- текущий статус;
- срок подачи;
- следующее действие;
- ответственный;
- признак просрочки.

### Базовые данные

- источник;
- ссылка;
- регион;
- тип обращения;
- вид работ;
- краткое описание;
- комментарий.

### Решение об участии

- дата запуска согласования;
- дата решения;
- итог;
- причина неучастия;
- комментарий.

### Обращение и папка

- номер обращения;
- ссылка на рабочую папку;
- дата создания папки;
- статус задачи.

### Затраты

- ответственный;
- срок;
- дата получения;
- статус проверки;
- дата утверждения;
- сумма затрат;
- сумма КП;
- плановая маржа;
- ссылка на расчёт;
- возвраты.

### Договор и протокол

- наличие проекта договора;
- статус анализа юристами;
- дата получения анализа;
- ключевые риски;
- нужен ли протокол;
- статус протокола;
- ссылка на файл.

### Документы

- статус комплекта;
- что не хватает;
- дата готовности;
- комментарий.

### КП и подача

- дата передачи затрат Кате;
- дата подготовки КП;
- сумма КП;
- дата направления на согласование МЛ;
- дата согласования МЛ;
- дата подачи;
- способ подачи;
- ссылка на подтверждение.

### Результат

- результат;
- дата результата;
- причина проигрыша;
- цена победителя;
- наша цена;
- вывод на будущее.

### Задачи

Таблица задач по заявке: тип, исполнитель, срок, статус, факт, просрочка.

### Ссылки

Список рабочих ссылок: папка, затраты, договор, протокол, КП, подтверждение подачи, обратная связь.

### Журнал событий

Хронологическая лента действий.

## 4. Мои задачи

Блоки:

- новые задачи;
- задачи в работе;
- просроченные задачи;
- задачи на сегодня;
- завершённые задачи.

Фильтры:

- заявка;
- тип задачи;

## 5. Справочники

Редактируемые справочники:

- статусы заявки;
- типы задач;
- причины неучастия;
- причины проигрыша;
- виды работ;
- источники обращений;
- внешние исполнители;
- способы подачи.

Минимально:

- пользователи;
- роли;
- базовые параметры системы;
- правила отображения сроков.
